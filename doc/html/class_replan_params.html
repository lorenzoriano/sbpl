double repair_time
 If this time out is not positive, it is ignored (this is the default). If the planner finds a first solution (in under maxTime of course) the time out for the planner becomes the minimum of maxTime and repairTime (clearly repairTime is useless if it is larger than maxTime). This second timer is useful because you often want a solution in a short amount of time (repairTime) where you are willing to let the planner improve the solution, but you are willing to wait a longer amount of time (maxTime) to get the first solution.
